# 📋 项目名称

**用药管理微信小程序**

## 📌 功能需求描述

1. **首页展示**

   - **本周用药统计**：显示用户本周的用药统计，包括用药次数、剩余药物等。
   - **最近一次用药**：展示最近一次的用药记录，包括药品名称、剂量、时间等信息。
   - **血药浓度曲线图**：

     - 基于用户输入的药品、剂量、体重等信息，绘制血药浓度随时间变化的曲线图。
     - **起始浓度计算公式**：每次用药后，根据以下公式计算起始浓度：

       \[
       \text{起始浓度} = \text{上一次残余浓度} + \left( \frac{\text{用药剂量}}{\text{体重}} \right) \times x
       \]

       其中，`x` 值默认为 2，但不同药品规格可微调。

     - **当前时刻的用药浓度计算**：
       - 获取系统当前时间。
       - 当前时刻的药物浓度 = **最近一次用药后的起始浓度** + **根据半衰期规则计算的浓度衰减后值**。
       - **半衰期规则**：每隔药物的半衰期时间，浓度衰减一半。

     - 用户可以为每种药品输入半衰期，并根据药品规格调整浓度变化。
     - 曲线展示：根据每次用药和浓度衰减，绘制随时间的浓度变化曲线。

2. **用药记录管理**

   - 用户可以录入新的用药记录，包括药品的品牌、规格、剂量、时间等。
   - 显示历史用药记录，支持按日期、药品种类等筛选记录。

3. **个人信息维护**
   - 用户可以录入和更新体重信息，体重数据与血药浓度计算相关联。

## 🗄️ 数据结构设计

### 用户数据（`users`）

- `userId`: 用户唯一标识
- `weight`: 用户体重

### 药品数据（`medicines`）

- `medicineId`: 药品唯一标识
- `name`: 药品名称
- `brand`: 药品品牌
- `spec`: 药品规格
- `halfLife`: 药品半衰期（单位：小时）
- `xValue`: 血药浓度计算中的系数，通常为 2

### 用药记录（`medRecords`）

- `recordId`: 记录唯一标识
- `userId`: 用户唯一标识
- `medicineId`: 药品唯一标识
- `dose`: 用药剂量
- `timestamp`: 用药时间戳

## ☁️ 云函数设计

1. **`addMedicineRecord`**：添加新的用药记录

   - 参数：`userId`, `medicineId`, `dose`, `timestamp`
   - 功能：将用药记录保存到数据库。

2. **`getWeeklyStats`**：获取本周的用药统计

   - 参数：`userId`
   - 功能：查询并计算本周的用药次数和剩余药物。

3. **`getLastRecord`**：获取最近一次用药记录

   - 参数：`userId`
   - 功能：返回用户的最近一次用药记录。

4. **`calculateConcentration`**：计算血药浓度随时间变化

   - 参数：`prevConc`, `dose`, `weight`, `xValue`, `halfLife`
   - 功能：根据公式计算血药浓度，模拟每隔半衰期浓度衰减，并返回浓度随时间的变化数据。

## 📌 前端页面结构

- **首页（`Home`）**：展示用药统计、最近一次用药、血药浓度曲线图。
- **用药记录列表页（`RecordList`）**：显示所有历史用药记录，支持筛选。
- **新增用药录入页（`RecordAdd`）**：允许用户录入新的用药记录。
- **血药浓度曲线页（`ConcentrationChart`）**：展示血药浓度随时间变化的图表。
- **个人信息页（`Profile`）**：维护和修改用户的体重信息。

## 📅 发布流程

1. **注册微信小程序账号**，获取 AppID。
2. **使用微信开发者工具**创建新项目，并开通云开发环境。

4. **上传并测试**，调试代码和云函数，确保功能正常。
5. **提交审核**，等待微信官方审核通过。
6. **发布上线**，审核通过后，将小程序正式发布供用户使用。

## ✅ 平台规则与开发逻辑校验（微信小程序）

> 结论：当前 `index.html` 仅用于 **Web 原型预览**，不能直接作为微信小程序代码发布。正式开发必须使用微信小程序原生技术栈。

### 1) 前端框架兼容性说明

- 微信小程序前端应使用：`WXML` + `WXSS` + `JavaScript` + 页面 `JSON` 配置。
- `Vue 3 / React` 生成的 Web 代码（如 `index.html` + 浏览器 DOM）**不直接兼容** 小程序运行时。
- 若使用跨端框架（如 Taro/uni-app），也需最终编译为小程序产物后再在开发者工具内运行。

### 2) 云开发能力映射

- **云数据库**：存储 `users`、`medicines`、`medRecords` 等集合。
- **云函数（Node.js）**：承担浓度计算、统计聚合、鉴权后写入等逻辑。
- **云存储**：存放头像、药盒图片、处方图片等资源。

```text

cloudfunctions/

```

### 4) 首页模块的小程序落地约束

- Home 页需通过 `Page({ data, onLoad, ... })` 组织页面逻辑。
- 图表可用小程序兼容图表库（如 `ec-canvas`）或 `canvas` 自绘；不能直接依赖浏览器专用 API。
- 时间、时区、日期建议统一以时间戳存储并在展示层格式化，避免统计口径偏差。

### 5) 发布与合规提醒

- 仅将小程序编译产物提交审核，Web 原型不可替代小程序包。
- 云函数涉及医疗相关信息时，需最小化采集并做好访问控制与日志审计。
